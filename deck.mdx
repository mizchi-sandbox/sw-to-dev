## Service Worker とは

合法ローカルプロキシ

```js
// client
navigator.serviceWorker.register("/sw.js");
// sw.js
self.addEventListener('fetch', event => {
  console.log('[fetch on sw]', event.request.url)
})
```

## 透過的 PWA の弱点

- SW とその他リソースは **並列で初期化される**
- 初回リクエストは SW でプロキシされる保証がない

## 対策

---

初期化を SW ready まで遅延する

```html
<!-- <script src="./main.js"></script> -->
<script type=module>
(async () => {
  await navigator.serviceWorker.register("/sw.js");
  await navigator.serviceWorker.ready;
  import("./main.js");
})()
</script>
```

---

新しい SW に更新されたらリロード強制

```jsx
const r = await navigator.serviceWorker.register("/sw.js");
navigator.serviceWorker.addEventListener("controllerchange", e =>
  window.reload()
);
setInterval(() => r.update(), 1000);
```

## 積極的 PWA

- SW でないと実現できない機能をふんだんに使う

## 積極的 PWA の弱点

- モダンブラウザのみ
- SEO 壊滅

---

## SW Side Tranform

- Babel / TypeScript
- Server Side GraphQL
- Express Emulator

---

## すべてが Hello World になる例

```js
self.addEventListener("fetch", event => {
  event.respondWith(new Response("console.log('hello world')"));
});
```

---

## Babel で返す

```js
self.addEventListener("fetch", event => {
  if (event.request.url.indexOf('.js') > -1) {
    event.respondWith(async () => {
      const res = await fetch(event.request)
      const raw = await res.text()
      return new Response(babel.transform(raw), {
        mode: "no-cors",
        headers: { "Content-Type": "text/javascript" }
      });
    });
  }
});
```
